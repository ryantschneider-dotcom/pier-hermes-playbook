Introduction
Overview & Purpose

A PIER offering site is not a simple property flyer. It is a fully designed, publicly accessible investment website that functions as a digital offering memorandum. It must be authoritative, data-rich, and persuasive — designed to qualify serious buyers and tenants before they ever pick up the phone.

Each site follows a consistent five-step process: gather the raw listing data, enrich it with independent market research, write copy that tells an investment story, lay out the page using PIER's design system, and deploy it to a custom subdomain under piercommercial.com.

Step One
Raw Data Intake & Foundation

Before writing any code or copy, you must establish a complete, verified factual baseline. Every piece of data on the site must trace back to a primary source.

Master Source Rule
Always treat Buildout.com as the absolute single source of truth for all property data. If any data point conflicts with what is in Buildout, Buildout wins. Do not use a figure from a news article or county record if it contradicts the active Buildout listing.

Collect the following fields before writing a single line of HTML or copy:

  • 1
    Address & Location
    Full street address, City, State, Zip, and County. This becomes the page title, the CNAME subdomain, and the hero headline.
  • 2
    Parcel ID (PARID)
    The county tax assessor parcel identifier. Used to cross-reference GIS data, assessed value, and lot boundaries. Example: Chatham County 20143-01002, Glynn County 03-10768.
  • 3
    Physical Specifications
    Total SF (always display as +-XX,XXX SF), acreage, year built, number of stories, construction type (masonry, tilt-wall, wood frame), and parking count.
  • 4
    Financial Data
    Asking price or lease rate, 2025 county assessed/appraised value, and annual property taxes. For leases, note the lease type (NNN, Gross, Modified Gross).
  • 5
    Zoning & Entitlements
    Current zoning code (e.g., B-C, B-N, MR, GC), by-right permitted uses, any conditional uses, and FEMA flood zone designation.
  • 6
    Images & Assets
    Collect all available property photos from Buildout. Never use AI-generated photos. Capture a Google Maps aerial screenshot with the site boundary outlined. Obtain the PIER logo from /home/ubuntu/upload/Brokeragetransp.png.
Public Site Exclusion Rule
You MUST strip these items from every public-facing site: Listing Agreements (commission, broker terms, protection period), Broker Opinions of Value / BOV (pricing ranges, value analysis), Buyer Target Lists, and any internal broker strategy notes. Run a grep check before publishing: grep -n "listing-agreement|BOV|Buyer Target|Commission" index.html

Step Two
Research & Data Enrichment

A PIER offering site does not merely list facts — it sells the location. The raw Buildout data tells you what the property is. Your research tells the reader why it matters. You must independently verify and enrich every site with market data, demographic context, and local growth narrative.

AlphaMap / Esri / U.S. Census
Pull 1-mile, 3-mile, and 5-mile demographic rings: Total Population, Total Households, Average HH Income, Average Age, Average Home Value, and Persons per Household.
Example: Williams Street — 83,318 residents within 3 miles, avg HH income $79,217 (AlphaMap).
Cushman & Wakefield MarketBeat
Query the most recent quarterly office, retail, or industrial MarketBeat report for the specific submarket. Extract current vacancy rate, average asking rent (NNN vs. Gross), and absorption trends.
Example: Williams Street — Savannah office vacancy 4.6%, avg asking rent $35.00/SF Gross (C&W Q2 2025).
LoopNet / CommercialCafe
Find 3-5 current comparable listings in the same submarket and property type to build a competitive comparison table showing how the PIER listing stacks up.
Example: Williams Street comparison grid showing PIER's $23.85/SF NNN vs. Class A downtown at $35.70–$39.84/SF Gross.
County GIS / Tax Assessor
Verify PARID, 2025 assessed value, lot dimensions, and exact acreage. Cross-check against Buildout. Also check for any active rezoning petitions or variance applications on the parcel.
Example: Glynn County PARID 03-10768 — 10.00 acres, last sold Dec. 2011 for $698,112.
Local News & Infrastructure Search
Search for recent capital investments, road projects, major retail/industrial openings, or rezoning activity within 3 miles of the property. This becomes the "Market in Motion" narrative section.
Example: YMCA site — Buc-ee's at Exit 42 (Q2 2025), Hillpointe 500K SF distribution center, $100M+ corridor investment.
Autodesk / RSMeans / Gordian
For investment sale sites, pull current construction cost per SF for the property type to calculate replacement cost and demonstrate the discount-to-replacement-cost investment thesis.
Example: Abercorn — National medical office avg $498/SF replacement cost (Autodesk/Cumming, May 2025).
Research Depth Guideline
Use a minimum of 8 independent sources for a standard listing. For major institutional or land sites, target 20+ sources. Always separate confirmed facts from inferences, and note when data is estimated vs. verified. Include source name, URL, and date in the site's footer disclaimer.

Step Three
Copywriting & Content Strategy

Write all copy in the "Golden Isles Prestige" tone: authoritative, corporate, and persuasive. Avoid encyclopedic dryness. Every sentence should either prove the investment thesis or advance the reader toward contacting Ryan. Avoid em-dashes in narrative descriptions.

  • 1
    The Hook — Hero Section
    Distill the entire offering into one powerful sentence that appears as the hero subtitle. Lead with the irreplaceable quality of the location or tenant. Example: "15,992 SF purpose-built medical office on Savannah's primary north-south business corridor — flanked by two major hospital systems, with income in place from HCA Healthcare."
  • 2
    The Investment Thesis — Overview Section
    Explain why this property matters beyond its specs. Highlight what makes it irreplaceable: the signalized corner, the high-credit tenant, the by-right density, the supply/demand imbalance. This is the "why buy" argument.
  • 3
    The Data — Spec Table
    Present raw data in a clean two-column spec table. Key metrics (SF, price, zoning, year built) should be displayed with the value in PIER orange using the highlight class. Use +- prefix on all SF figures.
  • 4
    The Proof — Market & Demographics
    Use the enriched research data to prove the thesis. For retail/office leases: show demographic rings and traffic counts. For investment sales: show market vacancy, comparable rents, and replacement cost. For land: show housing shortage data and recent capital investment.
  • 5
    Pricing Logic
    For investment sales (default): use "Pricing Available Upon Request" language to drive lead capture. For leases: state the exact rate (e.g., $23.85/SF/Yr NNN). Never display a price on a public sale site without Ryan's explicit instruction.
Pricing Withheld Language
"This property is offered for sale at a price to be disclosed to qualified buyers. To receive pricing information and the full offering memorandum, please contact the exclusive listing broker."

Step Four
Layout Logic & Design Rules

Choose one of two build architectures based on the complexity and prestige level of the listing. Both architectures share the same PIER brand system and section structure.

Architecture A — Standard
Single-Page HTML
Use For Standard sales, leases, flex/office, retail
Examples 6301 Abercorn, 2812 Williams St
Stack Pure HTML + CSS + Vanilla JS (no framework)
Fonts Playfair Display (headings), Inter (body)
Images Stored in web_images/ folder in repo root
Deploy GitHub Pages from main branch + CNAME file
Architecture B — Institutional
React + Vite + Tailwind
Use For Large land, institutional, complex multi-section sites
Examples Brunswick YMCA (10-acre redevelopment)
Stack React 19 + Vite + Tailwind 4 + shadcn/ui
Fonts Cormorant Garamond (headings), Lato (body)
Images CDN URLs via manus-upload-file --webdev
Deploy webdev_save_checkpoint → Publish button in UI

Standard Section Order — both architectures follow this sequence:

#hero Hero Full-viewport image/video, address title, stats bar, CTA buttons
↓
#overview Property Overview Investment thesis narrative + spec table (two-column layout)
↓
#highlights Investment Highlights 3-column card grid, orange top-border accent, key selling points
↓
#gallery Photo Gallery 3-column grid with featured span-2 hero image, lightbox on click
↓
#location Location & Context Google Maps embed + neighbor/amenity list with orange dot markers
↓
#market Market & Demographics 3-column demographic ring cards + market comparables table
↓
#contact Contact Orange background section, white broker card, Ryan's contact details

Design Element Rule
PIER Logo Use Brokeragetransp.png. Place on dark semi-transparent background. Never apply CSS filters (brightness, invert) — the PNG is already orange on transparent. No Filter
Section Identifiers Never use emoji or icon libraries as section markers. Use orange bullets, thin orange horizontal rules, or typographic hierarchy only. No Icons
Stats Bar Must appear immediately below the hero section. Contains 4-5 key metrics in bordered boxes on a solid orange background. Required
Heading Wrap All heading text must autosize to fit on a single line. Prevent wrapping to a second line using clamp() or white-space: nowrap where needed. Required
SF Display Always display square footage with a +- prefix (e.g., +-15,992 SF) to indicate approximate measurement. Required
Photos Never use AI-generated photos. Use only actual property photos from Buildout or verified aerial/street imagery from Google Maps. No AI Photos
Maps Always use a real street map (Google Maps embed or screenshot). Always include the PIER logo in the map caption or overlay. Required

Step Five
Deployment & URL Setup

All PIER offering sites are hosted on GitHub under the ryantschneider-dotcom organization and served via GitHub Pages or Vercel. Custom subdomains under piercommercial.com are mapped using a CNAME file in the repository root.

URL Naming Convention
Subdomain format: {streetaddress}.piercommercial.com
Examples: 6301abercornstreet.piercommercial.com · 2812williams.piercommercial.com · ymca.piercommercial.com
Keep it short, memorable, and address-based. No hyphens unless the address itself contains one.

Architecture A (HTML) — Deployment Steps:

  • 1
    Create GitHub Repository
    Create a new private repo under ryantschneider-dotcom using the naming pattern pier-{address}-offering (e.g., pier-6301-abercorn-offering). Use gh repo create --private.
  • 2
    Create the CNAME File
    In the repository root, create a file named exactly CNAME (no extension). The file must contain only the target subdomain on a single line, with no trailing slash. Example content: 6301abercornstreet.piercommercial.com
  • 3
    Place Assets in web_images/
    Store all property photos, aerial images, floor plans, and PDFs in a web_images/ subfolder. Reference them in HTML as relative paths: src="web_images/exterior_front.jpg".
  • 4
    Push to Main Branch
    Commit all files and push to the main branch. GitHub Pages will automatically serve the site from the root index.html.
  • 5
    Enable GitHub Pages
    In the repo Settings → Pages, set the source to the main branch, root folder. GitHub will detect the CNAME file and route traffic from the subdomain automatically.

Architecture B (React) — Deployment Steps:

  • 1
    Initialize with webdev_init_project
    Use the webdev_init_project tool with scaffold type web-static. Project name pattern: {property-slug}-offering. This registers the project and activates the Publish button in the Management UI.
  • 2
    Upload Images to CDN
    Run manus-upload-file --webdev /path/to/image.png for every image asset. Use the returned Cloudfront CDN URL in all <img src="..."> tags. Never use local file paths in React builds.
  • 3
    Save Checkpoint
    Run webdev_save_checkpoint to finalize the build and activate the Publish button in the Management UI header.
  • 4
    Publish & Assign Custom Domain
    Ryan clicks Publish in the Management UI. Then go to Settings → Domains and assign the custom subdomain (e.g., ymca.piercommercial.com).
CNAME File — Exact Format
# File: CNAME (no extension, repo root) # Contains exactly one line — the target subdomain 6301abercornstreet.piercommercial.com
If a Buildout Link Is Not Available
If a website link for a listing is not supplied, do not insert a random URL. Replace the missing link with the Buildout listing address. The property ID for the Buildout URL can be obtained from within the building information in Buildout.

Common Pitfalls & Fixes
Problem Cause Fix
Logo appears as white squares on light background CSS filter applied to logo image Remove all CSS filters from the logo <img>. The PNG is already orange on transparent. Use a dark backdrop instead.
Google Maps loads twice (console errors) Duplicate script injection in React Use singleton _mapsScriptPromise pattern. The loadMapScript() function must check if the script is already loaded before injecting.
Published site shows internal sections Forgot to strip private content before checkpoint Run grep check: grep -n "listing-agreement|BOV|Buyer Target|Commission" before every publish.
Images missing after React build publish Local file paths used instead of CDN URLs Always run manus-upload-file --webdev for every image. Never use /home/ubuntu/... paths in React code.
New React site has no Publish button Project not registered via webdev_init_project Must use webdev_init_project to register. Manual pnpm dev copies produce temporary URLs with no Publish button.
Heading text wraps to second line Fixed font size on long address strings Use font-size: clamp() or white-space: nowrap with overflow: hidden; text-overflow: ellipsis to keep headings on one line.
AI-generated photo used for listing No photo available from Buildout Request photos from Ryan. Use Google Street View or Maps aerial as a placeholder. Never generate a fake property photo with AI.
